# AGENTS.md — LLM Secrets Leak Detector

This file gives AI coding agents the context they need to work safely and effectively in this repository.

## Agent TL;DR
- **Target RE2 Compatibility.** Favor `google-re2` for regexes to ensure linear-time matching and prevent ReDoS.
- **Maintain Pattern Integrity.** Use the tools in `tools/` to manage the pattern library (normalization, deduplication, test data generation).
- **Verify with Acceptance Tests.** Always run `pytest tests/test_acceptance.py` to ensure core features like `--force-scan-all` and obfuscation modes work.
- **No Real Secrets.** Never include real credentials in the codebase or tests. Use synthetic data or base64-encoded samples.

## 🏗️ Core Philosophy & Architecture
- **Layered Detection**: Combines Pattern Matching (Regex), Entropy Analysis, and Contextual Heuristics.
- **Performance First**: Uses Aho-Corasick for keyword pre-filtering and RE2 for safe regex execution.
- **Privacy by Design**: All processing is local. Obfuscation (Redact, Hash, Synthetic) allows safe data sharing.
- **Streaming Support**: Optimized for line-by-line processing of large logs and real-time streams.

## 🛡️ Code Health & Safeguards (Mandatory)
- **ReDoS Protection**: Any new regex MUST be validated with `tools/run_redoctor.py` and `tools/run_safe_regex.py`.
- **Schema Validation**: Run `tools/regex_lint.py` to ensure rules in `data/` match the required schema.
- **Deduplication**: Run `tools/deduplicate_rules.py` before adding many new patterns to avoid redundancy.
- **Code Health**: If CodeScene is configured, target a Code Health score of 10.0 and run safeguards before proposing commits.

## 🛠️ Expert Best Practices
- **Regex Patterns**: Use non-capturing groups where possible. Avoid backreferences (not supported by RE2).
- **Keyword Filtering**: Every rule should have relevant keywords to benefit from the Aho-Corasick optimization.
- **Obfuscation**: Use `obfuscator.py` for all masking logic. Synthetic mode uses the `Faker` library.
- **Entropy**: Entropy thresholds should be balanced using `tools/migrate_entropy_factor.py`.

## 🧪 Testing Guidelines
1. **Pytest Suite**: Use `pytest` for all tests.
2. **BDD Scenarios**: Acceptance criteria are defined in `tests/acceptance.feature` and implemented in `tests/test_acceptance.py`.
3. **Test Data**: `data/*/test_data.json` is automatically generated by `tools/generate_test_data.py`. Do NOT edit these manually; edit the generators or rules instead.
4. **Regression**: Add new test cases to `tests/test_new_rules.py` when introducing new detection patterns.

## ⚙️ Tooling & Maintenance
- **Pattern Migration**: `tools/migrate_patterns.py` normalizes external rule sets.
- **Test Generation**: `python tools/generate_test_data.py` MUST be run after rule changes.
- **Linting**: `python tools/regex_lint.py` catches syntax and schema issues.
- **Release Verification**: Run `./demo.sh` to verify real-world usage (force-scan, obfuscation, streaming).

## ⚠️ Sensitive Areas
- `data/`: Contains the core logic of the detector (rules). High impact on false positives/negatives.
- `detector.py`: Core matching engine. Performance regressions here impact the whole tool.
- `obfuscator.py`: Responsible for ensuring secrets are actually hidden. Mistakes here lead to leaks.
